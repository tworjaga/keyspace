## Before Asking for Help

Please check these items before reaching out:

1. **Read the documentation** - Your question might already be answered
2. **Search existing issues** - Someone might have had the same problem
3. **Check your version** - Make sure you're using the latest version
4. **Try the basics** - Restart the application, check your configuration
5. **Gather information** - Collect error messages, logs, and system info

## Information to Include

When reporting issues, please include:

### For Bug Reports
- Keyspace version

- Operating system and version
- Python version
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Error messages (full traceback if available)
- Screenshots (if applicable)

### For Performance Issues
- System specifications (CPU, RAM, GPU)
- Attack type and configuration
- Target type and hash
- Wordlist size (if applicable)
- Performance metrics (speed, memory usage)

### For Installation Issues
- Operating system and version
- Installation method (pip, package, source)
- Python version
- Error messages during installation
- Dependencies installed

## Security Issues

**DO NOT** report security vulnerabilities publicly.

Instead:
1. Email security@keyspace.example.com
2. Or use [GitHub Security Advisories](https://github.com/tworjaga/keyspace/security/advisories/new)

3. Include detailed description and reproduction steps
4. Allow time for fix before public disclosure

See [SECURITY.md](SECURITY.md) for our security policy.
